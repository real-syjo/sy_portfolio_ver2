구현 기능

Data Collection

다양한 분야의 연구 데이터 타입을 모두 수용하고 효율적인 관리를 목적으로한 기능입니다. 부모노드를 중심으로 하위 노드에 연관된 데이터를 단수 혹은 복수로 저장하여 탐색 속도와 관리의 효율성을 높힐 수 있었습니다.

1. 데이터의 부모 노드인 Collection, set, section에 상세정보를 입력하여 등록합니다.
2. 등록할 자식노드의 수가 복수면 Pack 단수면 Entry를 선택합니다. (Pack은 Entry의 묶음)
3. 등록할 데이터의 DataType을 선택한다. 기존의 DataType selector (데이터 타입에 따라 입력 폼을 다르게 불러오는)를 연동하여 다양한 데이터의 형태를 구분하여 저장 할 수 있도록 합니다.
4. 선택한 dataType에 맞게 데이터를 업로드합니다.
5. 하나의 컬렉션이 완성이 되고 fancytree.js를 통해 key값으로 데이터를 출력합니다.

* 제가 만든 카카오 OVEN 프로토타이핑 툴을 활용한 연구데이터 Data Collection 사용자 메뉴얼 입니다.

Data Collection

_data Modeling

다양한 형태의 연구 데이터 결과를 연관된 콜랙션으로 묶어 탐색이 쉽고 효율적 관리가 가능하도록 계층형 모델로 구현하게 되었습니다. Collection, Set, Section, Pack, entry 단위로 각 테이블에 부모 Key값이 외래키로 묶여 있고 상위 노드가 삭제 되면 하위 노드 전체가 삭제 되도록 구현했습니다.

앱 마켓 상세

연구자가 등록한 연구 데이터 정보를 어플리케이션 형태로 공유하는 페이지 입니다. 앱의 상세정보와 즐겨찾기, 평점등록, 후기작성, 앱 다운로드 등 한 상세페이지에서 여러 콘텐츠를 볼 수 있도록 구현하였습니다.
